# 功能详解

本章节详细介绍无极助手的各项功能。

## 自动打怪

### 工作原理

软件通过图像识别技术检测游戏画面中的怪物：

1. **截图捕获**：实时捕获游戏窗口画面
2. **模板匹配**：识别怪物特征
3. **目标选择**：选择最近的怪物
4. **自动攻击**：发送攻击指令

### 怪物识别

#### 预设怪物

软件内置了常见怪物的识别模板，开箱即用。

#### 自定义怪物

可以添加自定义的攻击/忽略怪物：

1. 点击"辅助工具" → "怪物管理"
2. 选择"添加攻击怪物"或"添加忽略怪物"
3. 截取怪物特征图片
4. 保存配置

### 攻击策略

#### 目标优先级

- 优先攻击距离最近的怪物
- 可配置忽略特定怪物
- 支持自定义攻击顺序

#### 攻击模式

- **单体攻击**：逐个击杀怪物
- **范围攻击**：使用群攻技能（需门派支持）

## 技能调度

### 技能配置

在"挂机设置"中配置技能：

1. 设置技能快捷键（游戏中）
2. 配置技能释放顺序
3. 设置技能冷却时间

### 智能调度

软件使用堆排序算法管理技能冷却：

- 自动计算技能冷却时间
- 优先释放冷却完成的技能
- 避免技能冲突

### 技能类型

支持多种技能类型：

- **攻击技能**：主要输出技能
- **增益技能**：BUFF类技能
- **回复技能**：回血回蓝技能
- **特殊技能**：门派特色技能

## 状态监测

### 死亡检测

#### 检测方式

通过像素检测识别角色死亡状态。

#### 死亡处理

可配置死亡后的行为：

- **自动回城**：自动回城复活
- **原地复活**：使用复活道具原地复活
- **无操作**：不做任何处理，等待人工处理

### 断线检测

#### 检测方式

识别游戏断线画面。

#### 自动重连

启用断线重连后：

1. 检测到断线画面
2. 自动点击重连按钮
3. 等待5-15秒
4. 继续监测

::: warning 注意
断线重连使用前台鼠标操作，操作期间请勿使用鼠标。
:::

### 窗口尺寸检测

启动时会检测游戏窗口尺寸：

- 支持的尺寸：1024x768、1280x720、1280x1024、1920x1080
- 不支持的尺寸会发出警告

## 辅助工具

### 截图工具

可以截取游戏画面：

1. 点击"辅助工具" → "截图"
2. 选择截图区域
3. 保存图片

### 怪物管理

管理自定义怪物配置：

- 添加攻击怪物
- 添加忽略怪物
- 删除怪物配置
- 导入/导出配置

### 预设管理

保存和加载挂机配置：

- 保存当前配置为预设
- 加载已保存的预设
- 删除预设
- 导入/导出预设文件

## 设置选项

### 基础设置

#### 窗口设置

- 窗口位置记忆
- 窗口大小调整
- 启动时位置

#### 挂机设置

- 攻击速度
- 攻击方式
- 门派选择

### 快捷键设置

自定义各功能的快捷键：

- 开始/暂停
- 停止
- 显示/隐藏

### 更新设置

#### 自动更新

- 启动时自动检查更新
- 定时检查更新（每30分钟）
- 更新策略：
  - **可选更新**：用户可选择是否更新
  - **建议更新**：建议用户更新
  - **强制更新**：必须更新才能继续使用

#### 手动更新

在"设置"选项卡中点击"检查更新"手动检查。

## 数据管理

### 数据存储位置

所有数据存储在 AppData 目录：

```
%APPDATA%\TLWuJiAssistant\
├── settings.json          # 软件设置
├── license.lic            # 授权文件
├── presets\               # 预设配置
│   └── *.json
├── specific_mons\         # 自定义怪物
│   └── monster_config.json
└── monster_screenshots\   # 怪物截图
    ├── attack\
    └── ignore\
```

### 数据备份

可以备份所有配置数据：

1. 点击"设置" → "备份数据"
2. 选择保存位置
3. 生成 ZIP 压缩包

### 数据恢复

从备份文件恢复数据：

1. 点击"设置" → "恢复数据"
2. 选择备份文件
3. 确认恢复

## 安全功能

### 反调试检测

软件内置反调试机制，防止被逆向分析。

### 授权验证

- 在线验证：通过服务器验证授权
- 离线验证：本地缓存授权信息
- 心跳检测：定期验证授权状态

### 多实例限制

防止同时运行过多实例：

- 默认最多2个实例
- 使用互斥锁和文件锁机制
- 自动清理僵尸进程的锁

## 性能优化

### 后台操作

软件使用后台操作技术：

- 不干扰前台操作
- 使用 PostMessage 发送按键
- 不移动实际鼠标（除断线重连外）

### 资源占用

优化后的资源占用：

- CPU：< 5%（空闲时）
- 内存：约 50-100MB
- 网络：极少（仅授权验证和更新检查）

### 启动优化

- 延迟加载重型模块
- 自定义启动画面
- 启动时间 < 3秒
