# 更新日志

## v1.0.7 (2026-05-11)

### 🔄 授权系统改造

#### 服务端验证
- **对接服务端API**：实现卡密激活和在线验证
  - 激活接口：`/api/license/activate`
  - 验证接口：`/api/license/verify`
- **心跳检测**：每60秒自动验证授权状态
  - 卡密被禁用后自动停止程序
  - 实时更新授权信息显示

#### 离线验证机制
- **离线宽限期**：服务器故障时自动降级到本地验证
- **智能降级**：服务器错误不增加离线验证次数，用户不受影响
- **业务错误处理**：卡密无效/过期等业务错误直接失败，清除授权

#### 激活窗口优化
- 移除机器码显示，简化激活流程
- 显示剩余时间和到期时间
- 激活成功后自动进入主界面

### 🐛 Bug修复

#### Win11 卡顿问题修复
- **问题**：部分Win11用户每分钟出现界面未响应
- **原因**：心跳验证时每次都重新生成机器码，执行subprocess命令
- **修复**：机器码缓存到类级别变量，只在首次获取时执行subprocess

#### API参数修复
- **问题**：激活时提示 `'key' is required`
- **原因**：API请求参数名错误（`card_key` → `key`）
- **修复**：统一使用正确的参数名 `key`

## v1.0.6 (2026-05-10)

### ✨ 新功能

#### 前台鼠标操作设置
- **新增"前台鼠标操作设置"区域**：位于挂机设置页面的死亡检测下方

- **状态实时显示**：状态标签显示当前监测状态（等待绑定/监测中/已关闭）

## v1.3.0 (2026-05-05)

### 🔄 软件更新功能

#### 版本号统一管理
- **新增 `version.py`**：集中管理 `APP_VERSION` 和 `APP_NAME`
- **一处修改全局生效**：`main.py`、`helpers/updater.py`、`build_pyarmor.py` 均从此文件读取版本号
- **打包文件名自动带版本**：输出 `WujiAssistant-v1.3.0.exe`

#### 自动检查更新
- **启动后自动检查**：程序启动10秒后静默检查更新
- **定时检查**：每30分钟自动检查一次
- **智能弹窗**：自动检查时仅强制更新才弹窗，可选/建议更新不干扰用户
- **手动检查**：设置页点击"检查更新"始终弹窗

#### 强制更新机制
- **服务端控制**：通过 `strategy` 字段区分 `optional`（可选）、`recommended`（建议）、`forced`（强制）
- **无法跳过**：强制更新时点击 X 无效，必须更新或退出
- **自动删除旧版**：强制更新下载完成后，点击"退出并使用新版"会删除旧版 exe 并启动新版

## v1.2.0 (2026-05-04)

### 🎨 自定义启动画面

- **新增启动画面模块**：`splash_screen.py`，基于 Tkinter Canvas 绘制古风武侠主题动画
- **4种风格可选**：墨韵山水（暗金）、剑意江湖（红金）、青云之志（青绿）、丹青水墨（浅色）
- **打包时选择风格**：运行 `build_pyarmor.py` 选择自定义启动窗口后可指定1-4号风格

### ⚡ 启动速度优化

- **延迟导入机制**：将 cv2、numpy、win32api、keyboard 等30+个重型模块从文件顶部移至 `_lazy_imports()` 函数
- **后台线程加载**：启动画面显示后，由 daemon 线程在后台加载重型模块，动画与加载并行
- **启动体验提升**：exe 启动后立即显示动画，消除了3-5秒的空白等待

### 📦 资源文件打包优化

- **dmp 文件打包进 exe**：将 `dmp/` 目录（40个文件，233KB）通过 `--add-data` 打包进 exe 内部
- **简化分发**：用户不再需要维护外部的 `dmp/` 文件夹，减少误删风险
- **为自动更新铺路**：后续更新只需替换单个 exe，无需处理 zip 和资源合并
